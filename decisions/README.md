# Decisions — Architecture Decision Records (ADRs)

Record significant design decisions: what was decided, why, and what alternatives were considered.

## Format

Use frontmatter: `title`, `number`, `date`, `status` (accepted | deprecated | superseded). See `_templates/` for ADR-style structure.

## ADRs to add

| Number | Title |
|--------|--------|
| 001 | Primary/Secondary Membership via Self-Referencing FK |
| 002 | Schema Naming Conventions (snake_case, denormalised names) |

Add new ADRs when a non-trivial architecture or schema decision is made.
